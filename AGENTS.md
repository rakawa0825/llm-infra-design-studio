# Agent Rules

These rules apply to all Codex and LLM agents working in this repository.

## Confidentiality Boundary

- Work only with public-safe, synthetic content in this repository.
- Never include confidential identifiers, private customer names, real person names, real IP addresses, real network diagrams, real meeting transcripts, or real design excerpts.
- Do not derive samples from private project repositories or adjacent working directories.

## Source Handling

- Preserve source traceability from every extracted fact to a listed source.
- Separate confirmed facts, assumptions, unresolved issues, and detailed-design handoff items.
- Do not invent requirements.
- Do not silently resolve contradictions.
- Do not promote assumptions into confirmed facts.

## Review Behavior

- Flag conflicts, missing source evidence, ambiguous ownership, and customer or vendor confirmation needs.
- Always mark human approval points.
- Treat production impact, scope commitments, risk acceptance, and customer-facing language as human-owned decisions.

## Output Expectations

- Use concise Markdown tables and checklists where possible.
- Keep samples synthetic and reusable.
- Make uncertainty explicit.
- Prefer reviewable intermediate artifacts over final-looking answers.
