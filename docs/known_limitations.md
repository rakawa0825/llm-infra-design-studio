# Known Limitations

## Current v0.1 Limitations

- Text-based artifacts only.
- Synthetic examples only.
- No real LLM execution pipeline.
- No production UI.
- No company-specific document rendering.
- No Word / Excel / PowerPoint output.
- No network configuration generation.
- No network diagram generation.
- No full detailed design automation.
- No real customer deployment.

## Workflow Limitations

- The current sample is a minimal lifecycle case.
- It does not cover all infrastructure design domains.
- It does not cover all requirement definition patterns.
- It does not cover full parameter decision workflows.
- It does not cover test planning, rollout planning, or operations handover yet.
- It does not automate final approval.

## Validation Limitations

- Validators check structure and consistency.
- Validators do not prove design correctness.
- Validators do not approve requirements.
- Validators do not validate real customer constraints.
- Validators do not replace expert review.

## Future Extension Areas

- richer customer hearing intake,
- multi-meeting replay validation,
- requirement definition expansion,
- basic design section generation,
- detailed design handoff expansion,
- parameter sheet candidates,
- test plan candidates,
- rollout plan candidates,
- operations handover,
- Word / Excel / PowerPoint artifact adapters,
- config generation,
- network diagram generation.

## Design Stance

The repository intentionally prioritizes lifecycle structure, traceability, and human approval boundaries before format rendering or production automation.
