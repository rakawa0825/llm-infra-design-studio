# Riley Brown Codex 2026 Video Study

## Source

- Title: `Codex Full Course 2026: The NEW Best AI Coding Tool`
- URL: `https://www.youtube.com/watch?v=KXIdYEdOPys`
- Video ID: `KXIdYEdOPys`

## Purpose

This case records a public-safe summary of a local video-study workflow. The local processing explored how a long-form Codex tutorial could be converted into internal study artifacts, reusable workflow patterns, and notes for an LLM-assisted infrastructure design portfolio.

The purpose was not to publish a transcript or a full translation.

## Processing Flow

The local workflow followed this sequence:

```text
VTT subtitle input
-> subtitle noise removal
-> timestamped transcript
-> 10 minute segments
-> segment summaries
-> concept extraction
-> Codex usage pattern extraction
-> enterprise SASE-style workflow mapping
-> OpenAI application and portfolio notes
```

## Artifact Types Generated Locally

Local-only artifacts included:

- original VTT copy,
- cleaned timestamped transcript,
- cleaned plain transcript,
- segment files,
- Japanese segment summaries,
- key concept glossary,
- Codex usage pattern notes,
- infrastructure workflow mapping,
- application readiness notes,
- final Japanese report,
- processing log.

These local artifacts were used as reference material only. Full transcript files and long translated content are intentionally not committed here.

## Lessons Learned

- Long-form videos are more useful when processed into chapter summaries, concepts, and workflow mappings than when translated sentence by sentence.
- Codex-style workflows should be treated as task-to-artifact loops with human review gates, not as one-shot generation.
- The reusable asset is the processing method, not the source transcript.
- For infrastructure design work, the same pattern can map field inputs, meeting transcripts, screenshots, existing design documents, and vendor answers into traceable study or design artifacts.
- Portfolio language should separate what the video says, what is inferred, and what can be applied to enterprise workflows.

## Public Repository Boundary

This example does not include:

- full VTT subtitle text,
- cleaned full transcript,
- full translation,
- long source excerpts,
- downloaded audio.

Only the reusable workflow, generic Codex instruction template, and short case notes are committed.
