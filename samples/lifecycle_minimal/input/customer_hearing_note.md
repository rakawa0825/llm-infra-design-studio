# Customer Hearing Note

## Source

- Source ID: `SRC-001`
- Source type: customer hearing note
- Scenario: branch network traffic handling
- Status: `DRAFT`

## Hearing summary

The customer wants branch site internet-bound traffic to be evaluated for security inspection through a cloud security service.

The customer also raised concern about exception traffic. Some traffic classes may need different treatment if inspection creates operational impact or if the traffic is used for monitoring, management, or service continuity.

The customer would like the design team to avoid unnecessary disruption to monitoring or management traffic. The team should clarify which traffic must be inspected, which traffic may be treated as an exception, and which details belong in detailed-design validation.

## Items raised

- Branch internet-bound traffic should be reviewed for security inspection.
- Exception traffic needs to be identified before final design language is written.
- Monitoring and management traffic should not be assumed to follow the same path until validated.
- Final approval ownership is unclear and must be confirmed before artifact reflection.

## Approval boundary

This note expresses customer intent and clarification needs. It is not final approved requirement language and must not be used as final design approval.
