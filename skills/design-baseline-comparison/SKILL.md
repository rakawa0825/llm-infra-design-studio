# Design Baseline Comparison

## Purpose

Compare new evidence with the existing design baseline.

## When to Use

Use after evidence registration and official source reconciliation.

## Inputs

- Existing design baseline
- Meeting evidence
- Official source reconciliation

## Process

1. Extract current baseline assumptions.
2. Compare new evidence against baseline statements.
3. Identify impacted artifacts.
4. Classify impact and approval needs.

## Outputs

- Baseline comparison notes
- Design impact list

## Quality Checks

- Baseline status is preserved.
- Proposed changes are not marked approved.
- Unresolved items remain visible.

## Failure Modes

- Replacing baseline with new evidence without approval.
- Hiding design impact as wording cleanup.

## Human Review Required

Required for baseline change decisions.
