# Information Gap Request

## Purpose

Create targeted requests for missing evidence needed before design decisions.

## When to Use

Use when meeting statements, official source excerpts, or baseline comparisons reveal insufficient evidence.

## Inputs

- Official source reconciliation
- Baseline comparison notes
- Meeting evidence
- Issue register

## Process

1. Identify missing or conflicting evidence.
2. Classify gap type and owner role.
3. Attach affected artifact and required timing.
4. Set status to open unless answered, deferred, or cancelled by human process.

## Outputs

- Information gap request table

## Quality Checks

- Each gap has an owner role.
- Each gap identifies affected artifact.
- Questions do not imply a preferred answer.

## Failure Modes

- Turning gaps into assumptions.
- Asking vague questions that do not affect a design decision.

## Human Review Required

Required before sending stakeholder-facing requests.
