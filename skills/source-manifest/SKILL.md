# Source Manifest

## Purpose

Create an inventory of source materials with traceability metadata.

## When to Use

Use at the start of every design cycle or when new inputs arrive.

## Inputs

- Source files
- Source owner
- Date received
- Source type

## Process

1. Assign a stable source ID.
2. Record title, type, owner, date, and summary.
3. Mark confidentiality and sample-safety status.
4. Identify related workflow steps.

## Outputs

- Source manifest table

## Quality Checks

- Every source has an ID.
- Every extracted requirement can reference a source ID.

## Failure Modes

- Missing source ownership.
- Mixing real and synthetic sources.

## Human Review Required

Required before source content is used as a baseline.
