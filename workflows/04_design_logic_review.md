# Design Logic Review

## Goal

Review whether extracted requirements form a coherent design direction.

## Input

- Requirements table
- Communication matrix
- Review comments

## Process

1. Check design dependencies.
2. Identify contradictions.
3. Separate assumptions from decisions.
4. Create handoff items.

## Output

- Design logic review notes
- Handoff items

## Quality Gate

No design decision is marked complete without human approval.

## Human Approval / Escalation

Escalate architecture, security, operational, and migration-impact decisions.
